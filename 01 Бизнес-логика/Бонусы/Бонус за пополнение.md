# Бонус за пополнение

Прогрессивный бонус: клиент пополняет баланс картой — и сверх внесённого получает **внутренние деньги** (bonus-карман), тем больше, чем крупнее пополнение. Бонус **невыводим** — только трата внутри сервиса. `type=topup_bonus` в [[Баланс|балансе]].

> [!important] Главное
> Внесённое картой → **real** (выводимо). Начисленный сверху бонус → **bonus** (невыводимо, тратится первым, сгорает по TTL). Чем больше сумма одного пополнения — тем выше процент бонуса.

## Прогрессивная шкала
Порог определяется по сумме **одного** пополнения. Значения — параметр страны/кампании (пример в €):

| Сумма пополнения | Бонус % | Пример | На балансе после |
|---|---|---|---|
| < 20 | 0% | 10 → +0 | real 10 / bonus 0 |
| 20–49.99 | 5% | 20 → +1.00 | real 20 / bonus 1.00 |
| 50–99.99 | 10% | 50 → +5.00 | real 50 / bonus 5.00 |
| 100–199.99 | 15% | 100 → +15.00 | real 100 / bonus 15.00 |
| ≥ 200 | 20% | 200 → +40.00 | real 200 / bonus 40.00 |

## Как начисляется
```
A = сумма пополнения картой
pct = процент по шкале для A
bonus = round(A * pct)
→ real  += A      (type=topup,        pocket=real)
→ bonus += bonus  (type=topup_bonus,  pocket=bonus)
```
Обе записи в `balance_transactions` ([[Финансы]]), одной транзакцией, идемпотентно ([[Ключевые решения]]).

## Свойства бонуса
- **Невыводим**: вывести можно только real (внесённое). См. [[Баланс]] §вывод.
- **Тратится первым**: при аренде/подписке списывается bonus → real → карта.
- **Сгорает по TTL**: срок жизни — параметр страны (`type=expiry`).
- **Учёт**: маркетинговый расход, признаётся в момент траты; на [[Франшизы и сплит|сплит]] не влияет.

## Лимиты и антифрод
- **Потолок бонуса за период** (день/месяц) — параметр, защита от абуза «накрутки» крупными пополнениями с последующим возвратом.
- **Chargeback/возврат пополнения**: реверс real **и** пропорционально начисленного бонуса (`type=reversal`); если бонус уже потрачен — гасим остаток, недостачу фиксируем как долг ([[Баланс]]).
- Бонус невыводим, поэтому «обналичить» пополнение-бонусом нельзя — основной риск закрыт самой моделью.

## Промо-режим
Кампанией можно временно: поднять проценты (напр. «удвоенный бонус выходного дня»), сдвинуть пороги или добавить верхний tier. Управление — [[Веб-Admin]] (Marketing).

## Примеры
- Пополнил **50 €** → real 50, bonus +5.00 (10%). Аренда 1.80 → спишется из bonus → bonus 3.20, real 50.
- Пополнил **200 €** → real 200, bonus +40.00 (20%). Вывести можно только 200 (real); 40 — только тратить.
- Пополнил **15 €** → real 15, bonus 0 (ниже первого порога).

> [!question] Открытые вопросы
> Накопительная вариация (бонус по сумме пополнений за период, а не за одно) — нужна ли? Конкретные пороги/проценты и TTL — значения параметров. Потолок бонуса за период. См. [[Открытые вопросы]].
