# Купоны

Маркетинговый инструмент: скидки, бесплатные минуты, пополнения баланса, рефералка. Полная логика — от анатомии купона до антифрода. Активация по коду или **QR-сканированию** в [[Клиентское приложение|приложении]]. Данные — схема `coupons` ([[Схема базы данных (SparqX)]]).

## 1. Анатомия купона
Купон = идентификатор + **эффект** + **кому/где доступен** + **сколько раз** + **когда действует**.

| Группа | Поля | Смысл |
|---|---|---|
| Идентификация | `code` (uniq), `type`, `requires_qr` | Код/QR, тип, обязателен ли скан |
| Эффект | `effect_type`, `effect_value` | Что даёт (см. §2) |
| Сегментация | `coupon_regions` (страна / город / партнёр) | Где и для кого валиден (см. §6) |
| Лимиты | `max_total`, `max_per_user`, `max_per_day`, `used_count` | Сколько раз можно (см. §7) |
| Срок | `valid_from`, `valid_until` | Окно действия (см. §9) |
| Адресность | `target_customer_id`, `partner_id`, `owner_customer_id` | Для Compensation / Merchant / Referral |
| Условие | `min_order_amount` (опц.) | Минимальная сумма аренды для применения |
| Состояние | `status` | draft / active / paused / expired / archived (см. §4) |

## 2. Эффекты
Купон несёт **один** эффект (Combo — исключение, см. ниже):

| `effect_type` | `effect_value` | Что делает | Куда применяется |
|---|---|---|---|
| `free_minutes` | минуты | Доп. бесплатные минуты (как grace, €0.00) | Тариф конкретной аренды ([[Флоу аренды]]) |
| `percent` | % | Скидка % на тариф | Цена аренды |
| `fixed` | сумма | Фиксированная скидка | Цена аренды (итог не ниже 0) |
| `balance_credit` | сумма | Зачисление на баланс | **bonus**-карман ([[Баланс]]), невыводимо |

> [!note] Скидка на цену vs зачисление на баланс
> `free_minutes` / `percent` / `fixed` уменьшают **цену одной аренды**. `balance_credit` кладёт деньги на [[Баланс|баланс]] (bonus) и тратится потом по водопаду. Combo может объединять оба класса (напр. «−20% на аренду + 5 € на баланс»).

## 3. Типы купонов
Тип задаёт **кому адресован** и **какие проверки** включены; эффект — отдельно (§2).

| Тип | Кому | QR | Особые проверки |
|---|---|---|---|
| General | Всем | Опц. | Только сегмент + лимиты |
| New user | Только 1-я аренда клиента | Опц. | У клиента нет `charged`-ордеров |
| Merchant/Brand | От партнёра/бренда | Всегда | `partner_id`, сегмент партнёра |
| Compensation | Конкретному клиенту | Всегда | `target_customer_id` совпадает |
| Combo | Пакет (несколько эффектов) | Опц. | Как General + несколько эффектов |
| Referral | По личной ссылке клиента | Всегда | `owner_customer_id`, антифрод (§8) |

## 4. Жизненный цикл
```
draft → active → (paused ⇄ active) → expired | exhausted → archived
```
- **draft** — создан в [[Веб-Admin]], ещё не выдан.
- **active** — действует (внутри `valid_from..valid_until` и `used_count < max_total`).
- **paused** — временно отключён вручную.
- **expired** — вышел срок; **exhausted** — исчерпан лимит `max_total`.
- **archived** — снят, остаётся в истории для статистики.

## 5. Флоу активации (redemption)
```
Клиент вводит код / сканирует QR
        │
        ▼
[Пайплайн валидации §7] ── не прошёл ──▶ отказ с причиной
        │ прошёл
        ▼
Атомарно: used_count += 1 (条件 used_count < max_total)
        │
        ├─ effect = balance_credit ──▶ +bonus на [[Баланс]] (type=coupon)
        └─ effect = скидка ──────────▶ привязка к аренде, цена пересчитывается
        │
        ▼
Запись в coupon_redemptions (customer, order?, discount_applied / amount_credited)
```

> [!important] Атомарность счётчика
> `used_count += 1` выполняется условным апдейтом (`UPDATE … WHERE used_count < max_total`) под блокировкой строки — чтобы на последнем использовании два параллельных запроса не пробили лимит. По духу — как идемпотентность платежей ([[Ключевые решения]]).

## 6. Сегментация
- **По странам — обязательно**: страна клиента должна быть в `coupon_regions`.
- **По городам / партнёрам — опционально**: для локальных и merchant-акций.
- Валюта эффекта (`fixed`, `balance_credit`) = валюта страны клиента; кросс-валютных купонов нет.

## 7. Лимиты и пайплайн валидации
Купон применяется, **только если проходит все** проверки по порядку:

| # | Проверка | Источник |
|---|---|---|
| 1 | `status = active` | `coupons` |
| 2 | Сейчас внутри `valid_from..valid_until` | `coupons` |
| 3 | `used_count < max_total` | `coupons` |
| 4 | Клиент не превысил `max_per_user` | `coupon_redemptions` по `customer_id` |
| 5 | Не превышен `max_per_day` (за сегодня) | `coupon_redemptions` по дате |
| 6 | Страна клиента ∈ `coupon_regions` | `coupon_regions` |
| 7 | Сумма аренды ≥ `min_order_amount` (если задан) | расчёт аренды |
| 8 | New user → нет прошлых `charged`-ордеров | `orders` |
| 9 | Compensation → `target_customer_id` = клиент | `coupons` |
| 10 | Referral → антифрод-проверки (§8) | §8 |

## 8. Реферальная программа (тип Referral)
Личная ссылка/код клиента (`owner_customer_id` — пригласивший). **Бонус получают оба** участника — в bonus-карман ([[Баланс]], невыводимо).

```
Пригласивший делится ссылкой
        ▼
Приглашённый регистрируется (привязка owner_customer_id)
        ▼
Первая ОПЛАЧЕННАЯ аренда приглашённого  ← триггер
        ├─▶ пригласивший: +bonus (type=referral)
        └─▶ приглашённый: +bonus (type=referral)
```
- **Триггер** — первая `charged`-аренда приглашённого (пустые регистрации награду не дают).
- Размер бонуса — параметр страны.

> [!warning] Антифрод рефералки
> Одна награда на пару пригласивший–приглашённый. Самореферал запрещён (один клиент). Контроль накрутки: лимиты `max_per_user`/`max_per_day`, отслеживание совпадений телефона/карты/устройства. Бонус **невыводим** (нельзя «обналичить» рефералку).

## 9. Срок жизни и сгорание
- **Окно купона**: вне `valid_from..valid_until` активация невозможна (статус → `expired`).
- **TTL бонусов**: зачисления `balance_credit`/`referral` **сгорают** по сроку (`type=expiry` в [[Баланс|балансе]]). Значение TTL — параметр страны.

## 10. Стекинг и порядок скидок
- **Купоны применяются только к арендам**, не к покупке подписок.
- **Один купон на аренду** (купоны между собой не суммируются).
- С подпиской **сочетается**. Порядок: **подписка → купон → водопад [[Баланс|баланса]]** ([[Подписки и тарифы]]).

```
база  = тариф (price_per_hour × часы, не ниже min_charge)
шаг 1 = база × (1 − discount_pct подписки)
шаг 2 = применить купон (free_minutes / percent / fixed), итог ≥ 0
→ списание по водопаду bonus → real → карта
```

## 11. Возвраты и взаимодействие с финансами
- **Скидочные купоны** (`free_minutes`/`percent`/`fixed`) уже учтены в цене аренды → уменьшают **базу [[Франшизы и сплит|сплита]]** (это снижение цены).
- **`balance_credit`** — маркетинговый расход платформы, признаётся при трате бонуса, на сплит не влияет.
- **Возврат аренды**: возвращается фактически списанная сумма ([[Баланс]], `type=refund`); активация купона **не откатывается** (кроме случаев фрода). Зачисленный `balance_credit` остаётся.

## 12. Инварианты
- `used_count ≤ max_total` всегда (гарантирует условный апдейт §5).
- На клиента не больше `max_per_user` активаций одного купона; не больше `max_per_day` в сутки.
- Один `coupon_redemptions` на факт применения; повтор запроса не создаёт второй (идемпотентность по `order_id`/ключу).
- Сегмент по стране обязателен — купон без региона неактивируем.

## 13. Edge-cases
| Ситуация | Поведение |
|---|---|
| Код истёк / `used_count = max_total` | Отказ с причиной (expired / exhausted) |
| Гонка на последнем использовании | Условный апдейт пропускает только одного |
| Страна клиента не в сегменте | Отказ |
| New-user купон у клиента с историей | Отказ |
| Сумма аренды < `min_order_amount` | Купон не применяется |
| Реферал без оплаченной аренды приглашённого | Награда не начисляется |
| Самореферал / совпадение карты-телефона | Блок начисления (§8) |
| Возврат аренды со скидкой | Возврат списанной суммы; активация не откатывается |

## 14. Схема (таблицы `coupons`)
| Таблица | Ключевые поля | Назначение |
|---|---|---|
| coupons | id PK, code uniq, type, effect_type, effect_value, status, valid_from, valid_until, max_total, max_per_user, max_per_day, used_count, min_order_amount, target_customer_id, partner_id, owner_customer_id, requires_qr | Купоны всех типов |
| coupon_regions | id PK, coupon_id FK, country_code, city_code (опц.), partner_id (опц.) | Сегментация |
| coupon_redemptions | id PK, coupon_id FK, customer_id, order_id (опц.), discount_applied, amount_credited, status, redeemed_at | История активаций |

> [!note] Расширение текущей схемы
> Базовые поля (`code`, `type`, `free_minutes`, `topup_amount`, `max_total`, `used_count`) уже есть. Поля `effect_type/effect_value`, окно `valid_*`, `max_per_user/day`, адресность и `status` — предлагаемое расширение под полную логику. Старые `free_minutes`/`topup_amount` ложатся в `effect_type=free_minutes|balance_credit`.

## 15. Примеры
- **New-user −100% на 1-ю аренду**: `percent=100`, новый клиент, аренда 1.80 € → к оплате 0.00 €. Сплит на 0.00.
- **Merchant «+5 € на баланс»**: `balance_credit=5.00` → +5.00 в bonus. Тратится по водопаду.
- **Combo «−20% + 3 €»**: аренда 6.00 € → −20% = 4.80 (сплит на 4.80) + 3.00 в bonus отдельно.
- **Referral**: приглашённый сделал 1-ю аренду → пригласившему и ему по 2 € bonus.

## 16. UI и роли
Создание, лимиты, сегментация, статистика использования — [[Веб-Admin]] (вкладка Marketing). Клиент видит купоны и кошелёк в [[Клиентское приложение]] (QR-скан, список бонусов).

> [!success] Зафиксировано
> Бонусы сгорают по TTL, на вывод real есть лимиты (значения — параметры, [[Баланс]]). Купоны применяются **только к арендам**, не к подпискам.
