# Станции и павербанки

Физические устройства сети: станции (боксы со слотами) и павербанки внутри них. Схема `devices` ([[Схема базы данных (SparqX)]]).

## Таблицы (схема `devices`)
| Таблица | Ключевые поля | Назначение |
|---|---|---|
| stations | imei PK, owner_id FK, status, last_seen, signal_rssi, country_code | Реестр всех станций |
| powerbanks | sn PK, charge_pct, voltage, current, temperature, status_code | Состояние каждого павербанка |
| powerbank_slots | id PK, sn FK, station_imei FK, slot_number, mechanism_index | Где сейчас павербанк (удаляется при аренде) |
| orders | id PK, sn FK, rented_from, returned_to, user_id, status | История аренд → см. [[Аренды]] |
| telemetry_log | id bigint PK, station_imei FK, raw_packet blob, created_at | Сырые пакеты `0x10`, партиции по месяцам |

> [!important] powerbank_slots — ключевая механика
> Запись удаляется при аренде и создаётся заново при возврате в **любую** станцию. Это обеспечивает аренду в одном месте и возврат в другом. См. [[Флоу аренды]].

## Телеметрия и статусы
Каждые 20 минут станция шлёт `0x10` (329 байт) с состоянием всех слотов. Аномальные статусы (`0x02`, `0x03`, `0x09`, `0x0A`) уходят в `alerts.queue`. Подробно — [[IoT-протокол]].
