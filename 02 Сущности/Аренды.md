# Аренды

Ордер аренды павербанка — от выдачи до возврата. Таблица `orders` в схеме `devices`.

## Поля (таблица `orders`)
| Поле | Тип | Описание |
|---|---|---|
| id | PK | Идентификатор ордера |
| sn | FK→[[Станции и павербанки\|powerbanks]] | Какой павербанк |
| rented_from | FK→stations | Станция выдачи |
| returned_to | FK→stations | Станция возврата (`null` пока на руках) |
| user_id | FK→[[Клиенты и франшизы\|customers]] | Клиент (мягкая связь, users DB) |
| status | enum | renting / returned |

## Жизненный цикл
1. Создаётся при выдаче, `returned_to = null` — ордер активен.
2. Клиент возвращает в любую станцию → `returned_to` заполняется, `status = returned`.
3. Закрытие ордера триггерит расчёт в [[Финансы|payments]] и [[Франшизы и сплит|сплит]].

Полный путь — [[Флоу аренды]]. История аренд видна в приложении (вкладка History, долги — красный фон).
