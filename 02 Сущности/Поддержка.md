# Поддержка

Мультиканальные тикеты с перепиской и вложениями. Схема `support`.

## Таблицы (схема `support`)
| Таблица | Ключевые поля | Назначение |
|---|---|---|
| ticket_categories | id PK, name, parent_id FK (self) | Категории и подкатегории |
| tickets | id PK, customer_id FK, category_id FK, channel, status, priority, assigned_to FK | Тикеты |
| ticket_messages | id PK, ticket_id FK, sender_type, sender_id FK, message | Переписка |
| ticket_attachments | id PK, message_id FK, file_url, file_type, file_size | Вложения (фото/видео/файлы) |
| ticket_status_log | id PK, ticket_id FK, old_status, new_status, changed_by FK | История смены статусов |

## Каналы и логика
- Каналы: In-app, Email, WhatsApp, Telegram.
- Автоназначение: `Cannot return` → High priority; `Technical` → по стране оператора; round-robin.
- UI: два синхронизированных вида — Kanban (5 колонок) и Inbox (3 панели). См. [[Веб-Admin]].
