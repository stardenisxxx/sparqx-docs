# Архитектура системы

Многослойная система: устройства шлют данные через IoT, бэкенд из микросервисов обрабатывает бизнес-логику, фронтенды показывают данные по ролям.

```
Станции (MQTT) + Приложения  →  API Gateway  →  10 микросервисов (K8s)
       источники данных          единая точка       бизнес-логика
                                                          ↓
        PostgreSQL · Redis · RabbitMQ · MinIO  ←  очереди и хранилища
```

## Слои
| Слой | Компоненты | Назначение |
|---|---|---|
| Клиенты | Станции (MQTT), приложения, пользователь | Источники данных |
| API Gateway | Nginx/Kong, JWT auth, rate limiting | Единая точка входа |
| IoT | Alibaba IoT MQTT Broker, MNS Queue | Сообщения от 10 000+ устройств |
| Backend | [[Микросервисы\|10 микросервисов]] в Kubernetes | Бизнес-логика |
| Очереди | RabbitMQ: telemetry, payments, splits, notifications, media, alerts | Асинхронная обработка |
| Кэш | Redis Cluster: MQTT токены (7д), статусы, локи | Производительность |
| Хранилище | MinIO — медиа, прошивки, вложения | Файлы |
| БД | [[Схема базы данных (SparqX)\|PostgreSQL 16, 6 схем]] | Основное хранилище |
| Мониторинг | Prometheus + Grafana, ELK | Метрики и логи |
| CI/CD | GitLab CI + Docker + K8s rolling update | Автодеплой |
