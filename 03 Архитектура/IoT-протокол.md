# IoT-протокол

Связь сервера со станциями: аутентификация устройства, MQTT-команды, телеметрия. Брокер — Alibaba Cloud IoT (MQTT, порт 1883).

## Аутентификация устройства
```
GET /api/iot/client/con?simUUID=&uuid={IMEI}&deviceId=0&ssl=1&sign={MD5}
```
Подпись: `MD5(deviceId=|simMobile=|simUUID=|uuid=)`. Токен живёт **7 дней** — кэшируется в Redis (см. [[Ключевые решения]]).

## MQTT-команды
| Команда | Направление | Назначение |
|---|---|---|
| `{"cmd":"check"}` | Сервер → Устройство | Запрос состояния всех слотов |
| `{"cmd":"popup_sn","data":"SN"}` | Сервер → Устройство | Выдача павербанка по SN |
| `{"cmd":"popup","data":"N","io":"0"}` | Сервер → Устройство | Выдача по номеру слота |
| `{"cmd":"reboot"}` | Сервер → Устройство | Перезагрузка MCU |
| `{"cmd":"push_version_publish"}` | Сервер → Устройство | OTA-обновление прошивки |
| `0x10` (329 байт) | Устройство → Сервер | Телеметрия каждые 20 минут |
| `0x31` (11 байт) | Устройство → Сервер | Результат выдачи по SN |
| `0x40` | Устройство → Сервер | Статус возврата |

## Статусы павербанка (`0x10`)
| Код | Статус | Алерт |
|---|---|---|
| 0x00 | Слот пустой | Нет |
| 0x01 | Норма | Нет |
| 0x02 | Аномалия заряда | Да → alerts.queue |
| 0x03 | Ошибка коммуникации | Да → alerts.queue |
| 0x09 | TypeC короткое замыкание | Да → alerts.queue |
| 0x0A | Не вернулся после выдачи | Да → alerts.queue |
